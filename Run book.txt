1. 🎯 Purpose of This Runbook

This runbook explains:

How to set up and run the system

How to resume failed scans

How to troubleshoot issues

2. 🏗️ System Overview (Operational View)
Scan Images → Generate CSV → Review CSV → Execute Actions → Verify Results
                     ↓
                Checkpoint (resume)

pip install -r requirements.txt

3.3 Fix Import Path (IMPORTANT)
$env:PYTHONPATH="src"

3.4 Verify Setup
python main.py --help

4. 📂 Directory Preparation

Create working directories:
mkdir output
mkdir output\checkpoints
mkdir output\logs
mkdir output\archives

5. 🔍 IMAGE SCANNING OPERATIONS
5.1 Basic Scan
python main.py scan-images `
  --targets C:\pic `
  --extensions .jpg .png `
  --output-csv output\scan_results.csv
  
5.2 Scan with Hashing (Recommended)
python main.py scan-images `
  --targets C:\pic `
  --extensions .jpg .png `
  --output-csv output\scan_results.csv `
  --hash
✔ Enables:

duplicate detection later
integrity verification

5.3 Scan with Checkpoint (Production Mode)
python main.py scan-images `
  --targets C:\pic `
  --extensions .jpg .png `
  --output-csv output\scan_results.csv `
  --hash `
  --checkpoint output\checkpoints\scan_progress.json
  
5.4 What Happens Internally
    1. Recursively scan directories
    2. Filter only image files
    3. Extract metadata
    4. Optionally compute hash
    5. Save progress to checkpoint
    6. Write CSV report
	
6. 🔁 FAILURE RECOVERY (CRITICAL)
Scenario: Scan Interrupted

Example causes:

system crash
Ctrl+C
power failure

Resume Scan
python main.py scan-images `
  --targets C:\pic `
  --output-csv output\scan_results.csv `
  --hash `
  --checkpoint output\checkpoints\scan_progress.json
✔ System will:
- read checkpoint file
- skip already processed images
- continue from last point

Checkpoint File Example

{
  "processed_paths": [
    "C:\\pic\\img1.jpg",
    "C:\\pic\\img2.png"
  ]
}

8. 🔁 DUPLICATE DETECTION

Run Duplicate Scan

python main.py duplicates `
  --targets C:\pic `
  --checkpoint output\checkpoints\dup_scan.json
  
Output Example  

group_id | hash | file_count
  - C:\pic\a.jpg
  - C:\pic\b.jpg
  
Strategy
 
Option 1: Keep one, delete others
Option 2: Move duplicates to archive folder
Option 3: Rename duplicates

9. 📦 DIRECT FILE OPERATIONS
9.1 Copy Images

python main.py copy-images `
  --sources C:\pic\a.jpg `
  --destination D:\images `
  --policy rename
  
9.2 Move Images

python main.py move-images `
  --sources C:\pic\a.jpg `
  --destination D:\archive `
  --policy hash-compare-skip
  
9.3 Duplicate Handling Policies

skip                 → ignore file
overwrite            → replace existing
rename               → add suffix
hash-compare-skip    → skip if same content

10. 📄 CSV-DRIVEN OPERATIONS (MOST IMPORTANT)
10.1 Copy from CSV

python main.py csv-action `
  --csv output\scan_results.csv `
  --action copy `
  --destination D:\reviewed_images `
  --policy rename `
  --output-csv output\copy_results.csv

10.2 Move from CSV

python main.py csv-action `
  --csv output\scan_results.csv `
  --action move `
  --destination D:\archive `
  --policy hash-compare-skip `
  --output-csv output\move_results.csv
  
10.3 Delete from CSV

python main.py csv-action `
  --csv output\scan_results.csv `
  --action delete `
  --output-csv output\delete_results.csv
  
10.4 Dry Run (SAFE MODE)

python main.py csv-action `
  --csv output\scan_results.csv `
  --action move `
  --destination D:\archive `
  --dry-run `
  --output-csv output\dry_run.csv

✔ No files modified
✔ Only simulation

16. 🚨 TROUBLESHOOTING GUIDE
Issue: ModuleNotFoundError

✔ Fix:
$env:PYTHONPATH="src"

Issue: Permission Denied

✔ Fix:

Run PowerShell as admin
Skip system paths

Issue: Slow Scan

✔ Causes:

hashing enabled
large files

✔ Fix:
Disable hash or parallelize (future improvement)

Issue: Duplicate not detected

✔ Ensure:
--hash enabled during scan

Issue: CSV action failed

✔ Check:
file exists?
path correct?
permissions?

17. 🔐 SAFETY BEST PRACTICES

✔ Always run dry-run first
✔ Keep original CSV backup
✔ Use hash-compare-skip for safety
✔ Never run delete on first attempt
✔ Use checkpoint for large scans

18. 📈 PRODUCTION BEST PRACTICES

✔ Use checkpoint always
✔ Log everything
✔ Separate input/output drives
✔ Archive before delete
✔ Use CSV as audit log
